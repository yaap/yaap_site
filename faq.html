General info for unofficial builders

Revert back to intree gps-utils and bootctrl. Refer to this.
generate keys using:
subject='/C=US/ST=State/L=City/O=Android/OU=Android/CN=Android/emailAddress=email@example.com' for x in releasekey platform shared media networkstack verity otakey testkey sdk_sandbox bluetooth nfc; do ./development/tools/make_key vendor/yaap/signing/keys/$x "$subject"; done
obviously replace info in subject with whatever. run everything in the root build dir, after source envsetup and lunch target
no need to change anything else - if you build a user variant and the files are there it'll sign for you.
If you want to allow bootloader re-locking implement a change similar to: this commit and gen the key using:
openssl pkcs8 -in vendor/yaap/signing/keys/releasekey.pk8 -inform DER -out vendor/yaap/signing/keys/releasekey.key -nocrypt
build and verify vbmeta using (unpack a whole fastboot zip there):
avbtool verify_image --image /path/to/vbmeta.img
afterwards you can generate pkmd-signed.bin for users to flash using:
avbtool extract_public_key --key vendor/yaap/signing/keys/releasekey.key --output pkmd-signed.bin
which is typically flashed as such: fastboot flash avb_custom_key pkmd-signed.bin
  1. Can't flash OTA updates not signed by us: No one can trick the updater into flashing a malicious zip via MITM attacks or other methods. Installing anything signed differently requires a clean flash.
  2. Flash files authenticity assured: You can be sure the files you receive come from a trusted source (your maintainer or us).
  3. Dangers of test-signed ROMs: Anyone can sign malicious apps with public test keys and more easily gain privileges that regular apps cannot obtain.
  4. Banking apps verify environment signatures: While some checks can be spoofed, nothing compares to the security of actual platform signing.
  5. System apps can’t be replaced by malicious versions: Keys will never match unless compromised, preventing unauthorized update of system apps.
  6. Unauthorized apps can’t run as system user: Even apps in priv-app using Magisk cannot run as system user unless signed with platform keys and requesting it in their manifest. (Note: This can be compromised by Xposed, which is risky.)

There are many more advantages, but these are the main ones.
